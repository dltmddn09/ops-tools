# build-timeline

빌드 브랜치 타임라인 뷰어. 브랜치/패치 이벤트만 입력하면
레인별 버전 구간과 색 전환 시점을 규칙에 따라 자동으로 그려주는 정적 HTML.

- 팀 공유 URL: https://dltmddn09.github.io/build-timeline/
- 매주 갱신되는 파일은 `data.js` 하나뿐입니다.
- 페이지의 **⚙ 편집** 버튼을 켜면 캘린더를 직접 클릭해서 브랜치/패치를 입력할 수 있습니다
  (레인 사이 빈 칸 클릭 = 브랜치 생성, 패치 행 빈 칸에 "패치"/"핫픽스"라고 입력 = 자동 인식,
  색칠된 구간 클릭 = 버전/색 수정).
- 편집 후 **⬇ data.js 다운로드** → 파일 경로를 Claude에게 알려주면 레포에 반영 후 push.
- 규칙과 데이터 스키마, 갱신 워크플로우는 [SPEC.md](SPEC.md) 참고.
